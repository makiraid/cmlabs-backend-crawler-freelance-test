Mobile-Friendly Test

Apa Itu Mobile-Friendly Test cmlabs?

Mobile-friendly test adalah salah satu SEO tools gratis yang disediakan oleh cmlabs. Alat ini berfungsi untuk melakukan pengujian mobile-friendliness suatu url dari website. Dengan memasukkan alamat website ke dalam kotak yang disediakan, Anda dapat mengetahui seberapa baik website tersebut pada tampilan mobile.

Pada beberapa tahun kebelakang, Google telah menetapkan bahwa semua website harus menerapkan mobile-first. Hal ini dikarenakan berkembangnya jumlah pengguna mobile dari waktu ke waktu.

Oleh karena itu, sebagai pemilik website, Anda wajib memastikan bahwa website Anda telah mobile-friendly, responsif pada semua perangkat, dan menyajikan user experience yang baik agar dapat menempati posisi tinggi dalam halaman hasil pencarian.

Halaman Web Tidak Bisa Dijangkau?

Jika tool Mobile Friendly Test tidak dapat membuka halaman, maka akan muncul eror yang menjelaskan masalah tersebut. Masalah akses termasuk ketersediaan sistem atau waktu henti situs.

Tool Mobile Friendly Test ini membuka halaman sebagai Googlebot (tidak menggunakan data Anda, tetapi sebagai Google). Karena itu, kemungkinan besar akan diblokir oleh dokumen robots.txt.

Halaman Berisi Elemen yang Tidak Bisa Di-Mount

Jika tes ini tidak dapat mounting semua elemen yang digunakan pada halaman, Anda akan mendapatkan notifikasi. Resources adalah komponen eksternal yang disertakan oleh halaman, misalnya, gambar, CSS, atau file skrip. Hal ini dapat terjadi karena beberapa alasan:

  • Elemen tidak dapat dimuat dalam jangka waktu yang tepat. Untuk masalah ini, coba jalankan pengujian lagi di lain waktu
  • Elemen tidak ada di area yang disebutkan (kesalahan 404).
  • Elemen ini tidak tersedia untuk klien yang tidak login.
  • Elemen tersebut diatur untuk memblokir Googlebot dengan file robots.txt. Jika resources itu penting dan ada di laman web Anda sendiri, Anda harus mengijinkan Googlebot; jika ada di situs web lain, Anda harus menghubungi admin halaman web tersebut dan memintanya untuk membuka blokirnya.

Hasil Pengujian Tidak Stabil / Masalah Pemuatan Halaman

Jika Anda memiliki elemen yang tidak bisa dimuat atau masalah pile-up lainnya, Anda mungkin melihat hasil yang sedikit berbeda setiap kali menjalankan pengujian. Hal itu disebabkan oleh pengaturan resource yang ditumpuk bisa berbeda selama pengujian.

Manfaat Memiliki Website Mobile-Friendly

Memiliki halaman web yang mobile-friendly adalah suatu keharusan jika Anda ingin memiliki peringkat tinggi. Berikut ini adalah daftar keuntungan memiliki situs web yang mobile-friendly:

  • Mendapatkan lebih banyak lalu lintas dari penelusuran seluler
  • Berpotensi untuk mendapat peringkat tinggi di SERP
  • Menghindari penalti dari search engine
  • Memberikan pengalaman pengguna yang baik

Tips Membuat Halaman Web Lebih Mobile-Friendly

Sekarang Anda telah mengetahui pentingnya memiliki website yang mobile-friendly, lalu bagaimana membuat halaman tersebut menjadi mobile-friendly? Berikut adalah beberapa tips untuk meningkatkan mobile-friendliness sebuah website.

Membuat Tampilan Website Responsif

Tentu, tampilan website pada mobile dan desktop akan berbeda dikarenakan ukuran layar yang berbeda. Oleh karena itu, website Anda sebaiknya memiliki tampilan responsif yang berarti harus dapat dengan baik ditampilkan pada berbagai ukuran layar.

Dengan menyesuaikan tampilan website, otomatis website Anda akan menjadi mobile-friendly. Tampilan yang konsisten antara layar desktop dan mobile sangatlah penting bagi kenyamanan pengunjung.

Menggunakan AMP

AMP (accelerated mobile page) adalah framework yang diluncurkan untuk membuat website mobile sederhana dan membuat laman tersebut menjadi lebih cepat diakses dengan mengurangi beberapa elemen yang dianggap berat. Dengan menggunakan teknologi ini, kecepatan website akan meningkat dan lebih mobile-friendly.

Tingkatkan Kecepatan Loading

Selain menggunakan AMP untuk meningkatkan kecepatan, Anda dapat mengurangi elemen-elemen tidak diperlukan yang dapat menghambat proses loading halaman. Untuk penjelasan lebih lengkapnya, baca juga cara meningkatkan kecepatan loading website.

Optimasi Gambar

Elemen visual seperti gambar harus dioptimalkan dikarenakan elemen ini menyumbang waktu loading suatu website. Anda dapat mengatur format, melakukan kompresi gambar, serta menambahkan alt-text dalam gambar.

Pilih Tampilan yang Tepat

Tampilan yang dimaksud termasuk juga pada penggunaan font, form sederhana, hingga pop-up dalam halaman website. Elemen-elemen tersebut harus dioptimalkan pada perangkat mobile dengan ukuran layar yang terbatas agar tidak mempengaruhi tampilan keseluruhan website.

Demikian pembahasan mengenai Mobile-friendly test tool dari cmlabs lengkap dengan manfaat dan cara mengoptimalkannya. Kunjungi halaman tool ini untuk memulai pengujian terhadap situs web Anda secara gratis.

Cara Menggunakan Mobile-Friendly Test

Agar Anda dapat menggunakannya dengan baik, kami telah merangkum langkah-langkah yang harus Anda lakukan dalam mengoperasikan alat ini. Simak selengkapnya di bawah ini.

Buka Halaman Mobile-Friendly Test tool

Sebagai langkah utama, tentunya Anda harus membuka halaman dari tool ini. Setelah itu, Anda akan melihat beberapa fitur yang tersedia seperti kolom URL, tampilan hasil, progress pengecekan, pratinjau seluler, riwayat, hingga penjelasan singkat mengenai tools ini.

Gambar 1: Tampilan halaman Mobile-friendly test dari cmlabs

Masukkan URL Website

Agar bisa melakukan cek mobile-friendliness suatu website, Anda harus memasukkan URL website pada field yang tersedia. Pastikan Anda telah memasukkan URL dalam format yang benar (e.g https://example.com). Selanjutnya, klik tombol "Cek URL" untuk memberikan perintah pada tool ini agar memulai pengecekan pada website tersebut.

Gambar 2: Cara memasukkan URL website untuk dilakukan pengecekan mobile-friendliness

Analisa Hasil Mobile-Friendly Test

Setelah Anda mengklik tombol Cek URL, Anda akan menunggu beberapa saat agar robot melakukan pengecekan. Anda dapat melihat progressnya pada progress bar di bawah field URL. Proses ini tidak memakan banyak waktu, Anda bisa langsung mengetahuinya di field hasil di bagian kiri halaman.

Halaman hasil akan menampilkan status mobile-friendliness dari URL yang Anda masukkan beserta isu yang ditemukan bila ada.

Gambar 3: Hasil cek mobile-friendly dari URL yang dimasukkan sebelumnya.

Dengan mengetahui hasil mobile-friendliness dari alat ini, Anda dapat meningkatkan performa saat ini. Oleh karena itu, perlu adanya perbaikan secara berkala untuk menjaga atau memperbaiki isu yang muncul.

Meninjau Tampilan Seluler

Selain menampilkan hasil mobile-friendliness, Anda juga dapat melihat bagaimana tampilan website tersebut jika diakses melalui seluler. Fitur ini memungkinkan Anda menganalisa tampilan UI/UX pada mobile view.

Gambar 4: Fitur pratinjau seluler pada Mobile-Friendly test

Lihat Riwayat Penggunaan Mobile-Friendly Test

Hal terakhir yang bisa Anda lakukan menggunakan Mobile-Friendly Test adalah melihat riwayat penggunaan tool ini. Tool ini memungkinkan And auntuk melihat semua riwayat URL yang pernah diperiksa menggunakan tool ini.

Dengan Adanya fitur ini, Anda tidak perlu mengetikkan kembali URL yang pernah dicek jika ingin melakukan pengecekan ulang. Anda juga dapat menghapus riwayat penggunaan menggunakan tombol "Hapus" pada bagian kanan fitur riwayat penggunaan.

Gambar 5: Fitur riwayat pengecekan pada Mobile-Friendly Test tool dari cmlabs

Login Setelah 5 Kali Penggunaaan

Beberapa alat gratis cmlabs membutuhkah proses login setelah pemakaian sebanyak 5 kali, Mobile-friendly Test adalah salah satunya. Jika Anda telah menggunakannya sebanyak 5 kali, sistem kami akan secara otomatis menampilkan pesan yang menyatakan bahwa Anda telah mencapai batas pemakaian.

Gambar 6: Peringatan pemakaian telah mencapai batas. Pada peringatan ini, terdapat tombol login yang mengarahkan pada login page.

Anda dapat melakukan login dengan mengklik tombol login pada pesan peringatan yang muncul. Selanjutnya, Anda akan diarahkan menuju halaman login cmlabs. Silahkan masukkan data yang dibutuhkan, atau Anda dapat melakukan login melalui Google.

Gambar 7: Halaman login cmlabs

Setelah Anda berhasil melakukan login, Anda akan dapat kembali menggunakan alat ini untuk memeriksa mobile-friendliness secara gratis. Dengan melakukan login, sistem dapat menyimpan data riwayat penggunaan tool ini dalam satu tahun terakhir.
