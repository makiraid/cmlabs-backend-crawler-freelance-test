Site Audit Report Details

In the summary of the audit results, we display a detail page containing a list of links that have been successfully crawled. You can access the report in more detail by pressing the settings icon in the Action column. Our system will redirect you to the audit report details page for each selected URL.

The detailed page audit report contains complete information ranging from internal and external links connected, the HTTP status code of the selected URL, as well as the number of critical issues and warning issues on the page.

In addition, there are several menus to check the HTML structure, a list of links owned by a URL, page performance, redirect links, structured data used, to keyword analysis of page content. In this guideline, we will cover each of the detailed menus of the audit report.

Basic​

The basic report presented contains information on SEO implementation on the audited pages. This menu is divided into several sections such as identifying meta tags on web pages, checking HTML that is not working correctly, displaying page scores, content duplication, total click depth, and checking metrics for SEO optimization.

Metatags provide metadata information from web pages that can make it easier for search engines to index your website. Overall, this report will help web owners know the current state of the web and any SEO issues that could be improved. More information will be discussed at the following link: Site Audit Result Detail Page (Basic).

Links​

In this section, you will see the state of the links contained in the selected URL. The types of links displayed are related anchor, image, link, canonical, meta, alternate, to redirect.

Internal links are important for SEO because they can help Google to index your website and understand its overall structure. Details of the discussion regarding successfully crawled links will be discussed in the following article: Site Audit Result Detail Page (Links).

Page Performance​

The page performance report displays the results of website testing that are categorized as good, need improvement, or poor. In addition, we also provide reports related to core web vitals (CLS, LCP, FID) as well as page timing of the audited websites.

Good page performance can make users stay longer on your page. Knowing website performance issues can help you to improve your SEO implementation and user experience. The audit report will be discussed in more detail at the following link: Site Audit Result Detail Page (Page Performance).

Users can also view page performance analysis results with Lighthouse which provides more complete data such as performance, accessibility, best practices, SEO scores, and PWA.

Redirection​

A URL redirect occurs when there are two redirects from the initial URL to the destination URL. We will list the links on your page including the redirect URL, along with the HTTP status code, and information on the state of the link, and see if there are any recurring redirects. For more information, access it via the Site Audit Result Detail Page (Redirection).

Web owners need this report to find out how many redirect links there are in each URL. Too many page crawls can make search engines fail to index. This of course can affect SEO and a website's ranking in search results.

On Content​

Web owners should pay attention to the keywords used in the page content. Make sure you don't offer stuff your content with keywords in order to maintain your ranking in search results. Search engines can detect content as spam if the keyword is overused. The ideal keyword density according to SEO experts is around 1-2%.

The information displayed in this report includes the number of single phrases, keyword frequency, keyword density, and the readability index of the content on the selected page. A detailed discussion of audit results related to the content of your website can be accessed through Site Audit Result Detail Page (On Content).

That is a brief guideline regarding detailed audit reports that can be displayed. Each metric in the report details will be discussed in more detail in a separate article. Check out the explanation for a better experience in viewing the page audit report.
